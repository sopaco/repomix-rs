# 1. 项目概述

`repomix-rs` 是一个用 Rust 重新实现的 [Repomix](https://github.com/yamadashy/repomix)，目标是把你整个代码库“打包”成一个对 LLM 友好的单一文件。和它的 JavaScript 原版不一样，这个版本以速度和安全性为前提设计，同时内置了两种使用姿势：你既可以把它当成一个命令行工具直接在终端用，也可以把它作为一个 MCP 服务挂进 AI agent，让 agent 自己调用打包能力。

当前版本是 `2.0.0`，处于活跃开发状态。CLI、库 API 和 MCP 服务已经可用，配置 schema 还存在未来变动的可能。

## 1.1 C1 系统上下文

```mermaid
graph LR
    U[用户\nDeveloper] -->|调用| Z[repomix-rs\nCargo Workspace]
    A[AI Agent\nClaude / Zed] -->|MCP stdio| Z
    Z -->|读取| FS[文件系统]
    Z -->|可选| Git[git 客户端\ndiff / log]
    Z -->|可选| SL[Secretlint 规则引擎]
    Z -->|生成| O[打包输出文件\nXML/MD/TXT/JSON]
    O -->|供给| LLM[LLM API\n(GPT-4o 等)]

    style Z fill:#4A90D9,color:#fff
    style O fill:#7ED321,color:#000
    style LLM fill:#F5A623,color:#000
```

把 `repomix-rs` 想象成一个**代码包的压缩打包机**：你把一个目录塞进去，它会一边扫描内容，一边用 tree-sitter 把代码浓缩成签名级别的摘要，一边拿 Secretlint 扫一遍有没有漏出来的密钥，最后吐出一份结构整齐、带上 token 统计的文件。你既可以在终端里手动按一下按钮，也可以让一个 AI agent 在旁边自动按按钮。

在它之外有两个世界需要留意。一个是**它依赖的外部世界**：操作系统文件系统提供原始材料，`git` 负责可选地追加“谁改了这里”的历史视角，Secretlint 提供安全规则，tiktoken-rs 提供 token 计数词表。再往外是**真正的消费者世界**：那份最终生成的输出文件会喂给大语言模型，这是 `repomix-rs` 无法控制也不属于边界的部分。

三种角色走进这个系统时会获得三种不同的体验。**普通开发者**开终端跑一条 `repomix . --style markdown`，拿到 Markdown 报告就直接粘进对话窗。**AI agent** 走 JSON-RPC 走 stdio，调用 `pack_codebase` 之类的四个工具后拿到一个小型结果摘要，不用接触任何文件路径。**安全审查者**则会盯着 `suspicious_files` 列表，确认没有意外泄露 API key 之后再批准把报告发给模型。

## 1.2 核心特性

`repomix-rs` 围绕“快、安全、可嵌入”三个词设计了几个关键能力：

**多格式输出**：默认 XML，也提供 Markdown、纯文本和 JSON。不同风格适合不同下游——XML 最便于程序解析，Markdown 最方便人类阅读，JSON 适合 CI 流水线消费。

**Tree-sitter 压缩**：这是这个工具最核心的卖点。它用 tree-sitter 解析 10 种语言的代码，把函数或类的**完整实现体**替换成**签名摘要**（参数列表加返回类型，实现体变成 `...`）。一次压缩能把一个几千行的实现文件压到几十行，却保留了调用者最关心的接口信息。支持的 10 种语言是 JavaScript、TypeScript、Python、Rust、Go、Java、C、C++、Ruby、PHP。

**Token 计数**：调用 tiktoken-rs，默认用 `o200k_base`（GPT-4o 系列的词表）。计数结果写进文件头部和报告里，让用户在发给模型之前就知道这条 prompt 要花多少钱。如果 tiktoken 的下载初始化失败（离线环境常见），系统会明确打印警告并降级到按空白分割的估算——这种估算对中文/日文/韩文等无空白分隔的语言偏差较大，文档里会特别说明。

**Git 感知**：可选地读取 `git diff`、`git log`，甚至按最近修改频率对文件排序。实现上通过调用系统 `git` 可执行文件完成，无需 Cargo feature；若目录不是 git 仓库或 `PATH` 上没有 `git`，相关步骤会打印警告并跳过，不会中断整个打包流程。

**安全扫描**：默认开启 Secretlint，对每个文件跑规则检查，一旦命中规则就放入 `suspicious_files`，最终从输出中剔除。不让任何泄露密钥的文件溜进 LLM 的输入。

**并行处理**：用 rayon 做 rayon 并行；file I/O 阶段用 Tokio。十个文件里 tree-sitter 调用和 token 计数会分散到 CPU 各核同时执行。

**双模式消费**：一条命令可以当 CLI，也可以当 MCP 服务。`repomix --mcp` 直接把整个打包能力以 JSON-RPC 协议暴露出去，挂在 Claude Desktop 或 Zed 之类的 agent 运行时旁边，不用额外装别的服务。

## 1.3 技术栈总览

| 层次 | 库 / 工具 | 为什么选它 |
|---|---|---|
| 异步运行时 | Tokio | file I/O + MCP stdio 网络 |
| 并行计算 | Rayon | 文件内容处理的 data parallelism |
| 代码解析 | tree-sitter + 10 语言 grammar | 跨语言的准确 AST 抽取 |
| Token 统计 | tiktoken-rs (o200k_base) | OpenAI 兼容的 token 计数 |
| 安全扫描 | Secretlint | 成熟的密钥规则集 |
| Git | 系统 `git` CLI | `std::process::Command` 调用，无额外 Rust 依赖 |
| CLI | clap 4 derive | 零样板参数解析 |
| MCP | rmcp 1.7 | 标准 MCP server 抽象 |
| 配置 | serde / serde_json | 双向 JSON 编解码 |
| 日志 | tracing + tracing-subscriber | 结构化、可过滤的日志 |
| 剪贴板 | arboard | 跨平台 OS 剪贴板 |
| 交互提示 | dialoguer + console | 终端里的 `--init` 向导 |
| 文件匹配 | globset / ignore / walkdir | glob 模式 + .gitignore 集成 |

## 1.4 业务价值

`repomix-rs` 解决的核心痛点是：**把一个大项目塞进 LLM 上下文窗口，既不丢结构信息，又不让 token 账单吓一跳**。

Tree-sitter 压缩让上下文成本直线下降，但保留了模块边界和函数签名，LLM 仍然能理解代码的组织方式。Secretlint 做了一道安全门，降低“把密钥发给第三方模型”的风险。MCP 服务模式则解决了最尴尬的衔接问题——之前开发者要么手动跑命令、要么写脚本，现在 agent 直接通过标准协议调用，不需要 shell，不需要路径拼接，不需要解析输出文本。

整个工具一片“单一职责”：它不写代码，不改代码，不运行代码。它只负责把代码转化成一份高密度、低噪声、有统计、可审计的文本快照。这份快照才是它真正的产品。
