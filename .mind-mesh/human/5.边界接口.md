# 系统边界接口文档

本文档覆盖了 repomix-rs 的全部对外接口——包括 CLI 命令、MCP 工具参数、配置参数和环境变量。无论你是通过终端使用它、还是通过 AI Agent 调用它、还是集成到 CI/CD 流水线中，这里都列出了你可以调节的所有旋钮。

---

## 命令行接口 (CLI)

repomix-rs 使用 clap 框架进行参数解析，支持长参数（`--style`）、短参数（`-v`）、标志（`--compress`）和计数值（`-vvv`）。

### 主命令：`repomix`

**描述**：将代码库打包成 AI 友好格式。无参数时默认处理当前目录。

**使用模式**：
```bash
repomix [OPTIONS] [ROOT]              # 打包本地目录
repomix --remote <URL> [OPTIONS]      # 打包远程仓库
repomix --init                        # 交互式创建配置文件
repomix --mcp                         # 启动 MCP 服务器
```

**参数**：

| 参数 | 类型 | 是否必须 | 默认值 | 含义解读 |
|------|------|---------|--------|---------|
| `ROOT` | PathBuf | 否 | 当前目录 | 要打包的目录路径。如果不指定，默认处理你当前所在的目录 |
| `--remote <URL>` | String | 否 | — | 远程 Git 仓库 URL。支持 https/http/git/ssh 协议。系统会自动 clone 到临时目录、打包、清理——你的本地文件系统不会被"污染" |
| `--include <LIST>` | String | 否 | — | 逗号分隔的包含 glob 模式（如 `"*.rs,*.toml"`）。**追加**到配置文件的 include 列表——不会覆盖已有的模式 |
| `--ignore <LIST>` | String | 否 | — | 逗号分隔的排除 glob 模式（如 `"target/**,tests/**"`）。同样是追加模式 |
| `--style <xml\|markdown\|plain\|json>` | Enum | 否 | xml | 输出风格——决定了 AI 怎么"看到"你的代码。XML 结构最完整（每个文件有 tokens/chars 属性标注），Markdown 人类可读性好，Plain 最节省 token，JSON 方便程序处理 |
| `--output <PATH>` | String | 否 | (自动调节后缀) | 输出文件路径。默认根据 style 自动调后缀（.xml/.md/.json/.txt） |
| `--compress` | Flag | 否 | off | 启用 tree-sitter 代码签名提取。去掉函数体，只保留签名——AI 专注于"代码做什么"而非"怎么写" |
| `--remove-comments` | Flag | 否 | off | 去除源码注释。减少 token 消耗——但如果注释中有重要的设计决策，AI 会错过这些信息 |
| `--remove-empty-lines` | Flag | 否 | off | 压缩空行。与 `--remove-comments` 配合使用效果最好 |
| `--line-numbers` | Flag | 否 | off | 给每行加上行号。方便 AI 在回答中引用具体行："请解释第 42-58 行的逻辑" |
| `--truncate-base64` | Flag | 否 | off | 截断过长的 Base64 数据。防止图片/证书被展开成大量无意义 token |
| `--copy` | Flag | 否 | off | 输出同时复制到系统剪贴板。在无头/SSH 环境静默失败 |
| `--include-empty-directories` | Flag | 否 | off | 在目录树中展示空目录。默认只显示有文件的目录 |
| `--top-files-length <N>` | usize | 否 | 10 | 报告中显示 Top-N 个 token 最多的文件。设为 0 禁用此功能 |
| `--split-output <TOKENS>` | u64 | 否 | — | 超过此 token 阈值时自动分片。应对 LLM 的上下文窗口限制 |
| `--header-text <TEXT>` | String | 否 | — | 自定义头部文本，放在输出最前面。可以用来注入系统提示词或项目背景 |
| `--instruction-file <PATH>` | String | 否 | — | 指令文件路径。文件内容追加到输出末尾，适合放详细的分析指引 |
| `--include-diffs` | Flag | 否 | off | 包含 git diff（工作区和暂存区）。需要 git 仓库 |
| `--include-logs` | Flag | 否 | off | 包含 git log。展示最近 50 条提交（可通过配置调整数量） |
| `-v,-vv,-vvv` | Count | 否 | off | 详细日志级别。`-v` = DEBUG，更多 v = TRACE |
| `--init` | Flag | 否 | — | 交互式创建 `repomix.config.json` 和 `.repomixignore`。已有文件会提示确认覆盖 |
| `--mcp` | Flag | 否 | — | 以 MCP 服务器模式运行，通过 stdio 使用 JSON-RPC 协议通信 |

**使用示例**：
```bash
# 最简用法
repomix .

# 复杂场景：远程仓库 + 压缩 + 去注释 + Markdown 输出
repomix --remote https://github.com/owner/repo --compress --remove-comments --style markdown --output analysis.md

# 大项目分片输出
repomix . --split-output 64000 --style xml --output huge.xml
```

## MCP 服务器接口

当以 `--mcp` 模式运行时，repomix-rs 暴露 4 个工具供 AI Agent 调用。所有工具使用 JSON-RPC 格式通过 stdio 通信。

### 工具：`pack_codebase`

**描述**：打包本地目录。返回包含文件数、token 数、输出路径、目录树的 JSON 对象。

**参数**：

| 参数 | 类型 | 是否必须 | 默认值 | 含义解读 |
|------|------|---------|--------|---------|
| `directory` | String | 否 | 当前目录 | 要打包的目录。Agent 可以指定任意路径 |
| `style` | String | 否 | xml | 输出风格。未知值（如 "yaml"）会被拒绝为 `invalid_params`，不会静默降级 |
| `compress` | Boolean | 否 | false | 启用 tree-sitter 压缩 |
| `include_patterns` | String | 否 | — | 逗号分隔的包含模式 |
| `ignore_patterns` | String | 否 | — | 逗号分隔的排除模式 |
| `top_files_length` | Integer | 否 | 10 | 报告中 Top-N 文件 |

**返回**：
```json
{
  "description": "Successfully packed 42 files (12345 tokens) from repository",
  "total_files": 42,
  "total_tokens": 12345,
  "output_file_path": "/tmp/repomix_mcp_xxx/pack.xml",
  "result": "{ ... PackMetrics JSON ... }"
}
```

### 工具：`pack_remote_repository`

**描述**：克隆远程仓库到临时目录并打包。临时目录在请求完成后自动清理。

**参数**：

| 参数 | 类型 | 是否必须 | 含义解读 |
|------|------|---------|---------|
| `url` | String | **是** | Git 远程 URL。会校验格式，无效 URL 被拒绝 |
| (共享参数) | — | — | 同 `pack_codebase` 的 style/compress/include_patterns 等 |

### 工具：`read_repomix_output`

**描述**：读取之前生成的输出文件的原始文本内容。

**参数**：

| 参数 | 类型 | 是否必须 | 含义解读 |
|------|------|---------|---------|
| `file_path` | String | **是** | 之前 `pack_codebase` 返回的 `output_file_path`。会验证路径合法性 |

### 工具：`grep_repomix_output`

**描述**：在输出文件中搜索正则匹配，返回匹配行及可选的上下文。

**参数**：

| 参数 | 类型 | 是否必须 | 默认值 | 含义解读 |
|------|------|---------|--------|---------|
| `file_path` | String | **是** | — | 输出文件路径 |
| `pattern` | String | **是** | — | 正则表达式。无效正则被拒绝 |
| `context` | Integer | 否 | 0 | 匹配行前后的上下文行数。设为 3 可以看"前后三行" |

## 配置结构

### 完整配置

以下 JSON 展示了 `repomix.config.json` 的完整结构和所有默认值：

```jsonc
{
  "input": {
    "max_file_size": 52428800      // 50MB，超过此大小的文件跳过
  },
  "output": {
    "file_path": "repomix-output.txt",
    "style": "xml",                // "xml" | "markdown" | "plain" | "json"
    "parsable_style": false,       // XML 中附加机器可读的 tokens/chars 属性
    "header_text": null,           // 输出最前面的自定义文本
    "instruction_file_path": null, // 追加到输出末尾的指令文件
    "file_summary": true,
    "directory_structure": true,
    "files": true,
    "remove_comments": false,
    "remove_empty_lines": false,
    "compress": false,
    "top_files_length": 10,
    "show_line_numbers": false,
    "truncate_base64": false,
    "copy_to_clipboard": false,
    "include_empty_directories": false,
    "include_full_directory_structure": false,
    "split_output": null,          // 设为 64000 启用 64K token 分片
    "token_count_tree": {
      "show_tree": false           // 在输出末尾显示按目录层级汇总的 token 树
    },
    "git": {
      "sort_by_changes": true,
      "sort_by_changes_max_commits": 100,
      "include_diffs": false,
      "include_logs": false,
      "include_logs_count": 50
    },
    "json": {
      "no_timestamp": false        // 关闭后 JSON 输出不含时间戳，可确定性比较
    }
  },
  "include": [],
  "ignore": {
    "use_gitignore": true,
    "custom_ignore": []
  },
  "security": {
    "enable_secretlint": true
  },
  "token_count": {
    "encoding": "o200k_base"       // cl100k_base / p50k_base 等
  }
}
```

### 配置加载顺序

配置从四个层级按顺序合并，后一层覆盖前一层：

1. **内置默认值** — `RepomixConfig::default()` 定义的返回值
2. **全局配置** — `~/.repomix/repomix.config.json`（共享给所有项目）
3. **项目配置** — `./repomix.config.json`（团队共享，版本控制）
4. **CLI/MCP 参数** — 临时覆盖，不持久化

## 集成建议

**在 Claude Desktop 中使用 repomix**：
将以下配置添加到 `claude_desktop_config.json`（macOS 路径：`~/Library/Application Support/Claude/claude_desktop_config.json`）：

```json
{
  "mcpServers": {
    "repomix": {
      "command": "repomix",
      "args": ["--mcp"]
    }
  }
}
```

此后 Claude Desktop 的每个对话都可以通过 MCP 协议直接调用 repomix 的打包能力。

**在 CI/CD 流水线中使用**：
推荐使用 JSON 格式输出，便于下游任务解析统计信息：

```bash
repomix --style json --output codebase.json .
```

生成的 `codebase.json` 可以被后续的脚本解析，提取 `total_files`、`total_tokens` 等指标用于监控或告警。

**在 Cursor 中使用**：
配置路径为：Settings → MCP → Add new global MCP server，填入与服务端 `repomix --mcp` 建立连接的 JSON 配置（同上）。
